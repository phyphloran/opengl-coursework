# opengl-coursework

Курсовой проект на OpenGL с визуализацией тора, цилиндра и линии их пересечения.

Сцена построена по мотивам варианта с изображением, где:
- зеленый объект является тором;
- красный объект является цилиндром;
- цилиндр смещен к краю тора;
- желтая линия показывает линию пересечения поверхностей;
- дополнительные каркасные сферы можно включать и выключать для демонстрации метода концентрических сфер.

## Что реализовано

- Отрисовка тора вручную через `GL_QUAD_STRIP`.
- Отрисовка цилиндра и его крышек через `GLUquadric`.
- Черный фон сцены.
- Перспективная проекция с корректным масштабированием при изменении размеров окна.
- Вращение камеры левой кнопкой мыши.
- Переключение вспомогательных сфер клавишей `E`.
- Линия пересечения тора и цилиндра, рассчитанная для текущего смещенного положения цилиндра.
- Освещение сцены с более мягкими тенями, чтобы формы оставались объемными, но не выглядели слишком темными.

## Управление

- `ЛКМ` + движение мыши: вращение камеры вокруг сцены.
- `E`: скрыть или снова показать вспомогательные сферы.

## Геометрия сцены

В проекте используются следующие основные параметры:
- большой радиус тора `g_torusMajorRadius = 7.0`;
- малый радиус тора `g_torusMinorRadius = 2.45`;
- радиус цилиндра `g_cylinderRadius = 2.85`;
- длина цилиндра `g_cylinderLength = 20.0`;
- смещение цилиндра вдоль оси `Z` задается константой `g_cylinderOffsetZ`.

Тор расположен в начале координат. Цилиндр ориентирован вдоль оси `X` и дополнительно смещен так, чтобы проходить рядом с внешней частью тора, как в референсном изображении.

## Как собрать

Проект рассчитан на Visual Studio 2022 и классический OpenGL под Windows.

Один из вариантов сборки:
1. Открыть `Prime.sln` в Visual Studio.
2. Выбрать конфигурацию `Debug | x64` или `Release | x64`.
3. Собрать проект.

Исполняемый файл после сборки появляется в каталоге:
- `x64\Debug\Prime.exe`

## Технические детали

- Используются библиотеки `opengl32.lib` и `glu32.lib`.
- Окно создается через WinAPI.
- OpenGL-контекст создается через `wglCreateContext`.
- Для корректного отображения при изменении размеров окна пересчитывается `aspect ratio` в `gluPerspective`.
- В проекте применяется orbit-камера: пользователь вращает сцену мышью, а камера смотрит на центр композиции.

## Примечание

Дополнительные сферы сохранены как наглядный вспомогательный элемент. После смещения цилиндра к краю тора они служат скорее иллюстрацией идеи метода, чем строгим геометрическим построением для текущей несимметричной конфигурации.
